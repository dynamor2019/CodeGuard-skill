# CodeGuard v2 验收清单（可打勾）

## A. 基础功能
- [ ] `guard` 可在 Lite/Standard/Strict 运行
- [ ] `commit --tx` 可批量确认并写入审计
- [ ] 旧命令兼容层可正常提示迁移
- [ ] `doctor lock` 能正确区分 occupied/stale
- [ ] `unlock` 需要显式确认才能执行

## B. 审计完整性
- [ ] 每个事务有唯一 `tx_id`
- [ ] `pending/approved/confirmed/expired` 状态可追踪
- [ ] 每个文件存在 `pre_hash/post_hash`
- [ ] confirmed 记录可追溯到审批人和时间
- [ ] 异常失败不会写入假成功记录

## C. 编码保护
- [ ] 编辑前记录 encoding/BOM/line-ending
- [ ] 编辑后自动校验三项一致
- [ ] 不一致时自动回滚并返回错误码
- [ ] 白名单路径需显式参数才允许变更编码

## D. 效能指标（2 周）

采样窗口：连续 14 天，至少 30 个开发任务，含核心模块任务 >= 5。

### 必测指标与阈值
- [ ] 任务耗时变化：P50 不高于基线 +10%，P90 不高于 +15%
- [ ] token 增幅：平均增幅 <= 20%，目标下降 30%-50%
- [ ] 阻塞率（含锁文件）：<= 3%
- [ ] 回滚成功率：>= 99%
- [ ] 缺陷逃逸率：不高于基线（目标下降 >= 10%）

## E. 两周试运行计划

第 1 周：
- [ ] 开启 `tier=auto`
- [ ] 每天汇总：锁冲突、pending 超时、编码回滚
- [ ] 修正高频错误文案

第 2 周：
- [ ] 对核心路径强制 Strict
- [ ] 对低风险任务固定 Lite
- [ ] 形成去留决策报告

## F. 决策门槛（继续/降级/下线）

继续（保留并推广）：
- [ ] 同时满足：阻塞率 <= 3%、回滚成功率 >= 99%、缺陷逃逸率不升高

降级（保留但降强度）：
- [ ] 任一触发：任务耗时 P90 > +20% 或 token 增幅 > 30%
- [ ] 动作：默认 Lite，Strict 仅核心模块

下线（暂停使用）：
- [ ] 任一触发：审计链断裂、回滚成功率 < 95%、阻塞率 > 8% 且连续 3 天

## G. 发布前确认
- [ ] 公告文案已发布
- [ ] 团队命令模板已更新为 `guard/commit`
- [ ] CI 示例已更新
- [ ] 值班人员已掌握 `doctor lock` 与应急回滚
